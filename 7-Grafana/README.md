# Домашнее задание №7 "Grafana"

**Цель:**

Научиться визуализировать технические и пользовательские метрики в Grafana, включая вычисление SLA.
Задание:
1) Создай дашборд с панелями:
   * Readiness probes: статус контейнеров oncall и mysql.
   * Ресурсы контейнеров: текущая загрузка CPU и RAM тех же контейнеров.
   * Дополнительные метрики: добавь 1-2 метрики, которые считаешь полезными для анализа состояния приложения. Для каждой метрики поясни, зачем она нужна.
   * Проберы (из ДЗ №7): визуализируй результаты работы своих проберов в отдельной панели.
   * SLA (из ДЗ №7): создай панель с рассчитанным SLA и укажи формулу расчета.

2) Продемонстрируйте просадку SLA

   Добейтесь контролируемого снижения SLA и покажите это на графике.
   
   Рекомендуемые варианты (но не ограничение):
   * временно остановить oncall (масштабировать реплики до 0);
   * сделать readinessProbe намеренно «падающим»;

**Подсказка:** Переключи организацию в Grafana на свою группу (меню в левом верхнем углу).

**Что сдать:**
* Ссылку на итоговый дашборд.
* Демонстрация просадки SLA - скриншот или ссылку на дашборд с Lock time range
* Описание добавленных метрик и их назначение.
* Формулу (методику) расчета SLA.

**Критерии оценки:**
* 4 балла — панели по контейнерам.
* 2 балла — панели по проберам.
* 4 балла — корректная панель с SLA.
