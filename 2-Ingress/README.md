# Домашнее задание №2 "Ingress"

В рамках данного задания Вы попрактикуетесь в управлении трафиком до приложений в Kubernetes. Вам необходимо написать и применить манифесты по следующим требованиям:

* должно быть развёрнуто приложение **hello-app** на базе образа **gcr.io/google-samples/hello-app:1.0**
* должно быть развёрнуто приложение **whoami** на базе образа **nginx:1.12**
* при запросе по любому пути приложение **whoami** должно возвращать HTTP-ответ со статус-кодом **200** и текстом формата **"You are <Ваше имя> from <Ваша группа>"**. Подсказка: детальнее изучите манифесты пункта №5 из примеров домашнего "Основы Kubernetes".
* приложение hello-app должно быть доступно по адресу **st-\<user>.ingress.sre-ab.ru/hello**, где \<user> - Ваш пользователь из kubeconfig
* приложение whoami должно быть доступно по адресу **st-\<user>.ingress.sre-ab.ru/whoami**, где \<user> - Ваш пользователь из kubeconfig
* в Ingress должен быть настроен таймаут на установление подключения к приложению равный 10 секундам
* в Ingress должен быть настроен метод балансировки **round robin**.

Примеры настройки параметров балансировки Nginx Ingress можно найти в документации.

**Важно!**

Для обеспечения возможности запроса по указанным адресам из браузера вам необходимо добавить локальную DNS-запись:
> 10.128.0.32 st-<Ваш пользователь>.ingress.sre-ab.ru
* в файл **/etc/hosts** для MacOS / Linux
* в файл **C:\Windows\System32\drivers\etc\hosts** для Windows

В качестве ответа на задание приложите архив, включающий:
* скринкаст или скриншоты, демонстрирующие процесс развёртывания, развёрнутые ресурсы и доступные по адресам страницы в браузере;
* созданные манифесты, использованные для развёртывания приложений
